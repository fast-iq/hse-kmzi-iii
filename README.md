# hse-kmzi-iii

## Практическая работа 8

## Схемы электронной подписи

3 ЗАДАНИЕ
1)	написать программную реализацию схемы электронной цифровой подписи, представленной в ГОСТ Р 34.10-2012. Программная реализация должна быть выполнена студентом самостоятельно без использования готовых библиотечных решений (допускается использование готовой реализации хэш-функции ГОСТ Р 34.11-2012);
2)	подготовить отчет о выполнении работы.

Программа должна обладать следующей функциональностью:
1)	принимать на вход файл, для которого необходимо сформировать или проверить электронную цифровую подпись;
2)	принимать на вход файл, содержащий электронную цифровую подпись;
3)	принимать на вход ключ подписи или ключ проверки подписи;
4)	давать пользователю возможность сгенерировать ключевую пару;
5)	осуществлять формирование или проверку электронной цифровой подписи по выбору пользователя.

## Запуск
### Реализация ГОСТ Р 34.10-2012 (ЭЦП)
#### Использование:
 - go run main.go generate --priv <файл> --pub <файл>
 - go run main.go sign --in <файл> --priv <файл> --sig <файл>
 - go run main.go verify --in <файл> --pub <файл> --sig <файл>
